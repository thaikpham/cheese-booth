# Cheese Booth v0.1.4

project of colorlabv2

`v0.1.4` tập trung vào độ rõ ràng khi vận hành kiosk và độ ổn định của luồng cài đặt nội bộ qua GitHub Releases.

## Điểm mới

- Thêm cảnh báo rõ ràng để phân biệt `Browser Preview` với `Desktop Tauri`, giúp tránh nhầm bản preview là bản kiosk desktop thật.
- Thêm trạng thái hướng dẫn khi desktop đã mở nhưng chưa chọn thư mục lưu, để người vận hành biết cần vào `Settings > Thư mục lưu` trước khi chụp.
- Bổ sung hiển thị trạng thái `Runtime` và `Auto-save local` trong dashboard để kiểm tra nhanh ngay trên máy kiosk.

## Cải thiện phát hành nội bộ

- Rà soát lại toàn bộ installer snippets cho macOS, Windows, Ubuntu và Fedora.
- Siết pattern nhận diện asset để bám đúng naming convention của GitHub Release workflow: `colorlabv2-photokiosk-[platform]-[arch][ext]`.
- Giảm rủi ro script cài nhầm asset hoặc fail do tên file release thay đổi theo nền tảng.
- Thêm checklist publish `v0.1.4` vào README để chuẩn hóa quy trình tag và xác nhận asset sau khi GitHub Actions hoàn tất.

## Ghi chú vận hành

- Auto-save local chỉ hoạt động trong app desktop Tauri.
- Nếu đang mở bằng trình duyệt, app chỉ dùng để preview giao diện và camera.
- Sau khi cài desktop app, cần chọn thư mục lưu trước khi chụp để ảnh và boomerang được ghi xuống máy local.

## Xác nhận trước phát hành

- `npm run lint`
- `npm run build`

## Installer mong đợi trong release

- macOS Apple Silicon: `.dmg`
- macOS Intel: `.dmg`
- Windows x64: `.msi`
- Windows ARM64: `.msi`
- Ubuntu x64: `.deb`
- Fedora x64: `.rpm`
